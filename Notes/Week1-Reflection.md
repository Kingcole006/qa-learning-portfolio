## 💡 Key Takeaways

### Most Important Lesson
**"Testing is about thinking critically, not just clicking buttons."**

The shift from "does it work?" to "how could this break?" was fundamental. Learning to anticipate edge cases and user behaviors that developers might not consider.

### Biggest Surprise
**How detailed good bug reports need to be.**

I initially thought "it doesn't work" was enough. Learning about:
- Steps to reproduce
- Expected vs actual results
- Environment details
- Screenshots/evidence
- Severity classification

This level of detail makes the difference between a bug getting fixed quickly vs ignored.

### Most Challenging Concept
**Edge case discovery**

Going from obvious test cases (happy path) to thinking about boundary conditions, invalid inputs, and unusual user behaviors required a mental shift.

**Example:**
- Basic: Test login with valid credentials
- Edge cases: Empty password, SQL injection attempts, very long usernames, special characters, spaces

## 🚧 Biggest Challenge

**Thinking comprehensively about edge cases.**

It's easy to test the obvious scenarios. The challenge is systematically thinking through all the ways something could fail or be misused.

**How I overcame it:**
- Used frameworks (boundary testing, equivalence partitioning)
- Asked "what if..." questions repeatedly
- Reviewed examples of real-world bugs
